The Django Project - Restaurant Lookup

This new Django web application bridges the gap between restaurants and users seeking convenient food ordering. Users can browse menus and make informed choices, while restaurants gain a wider audience, analyze user data to optimize menus, and grow their business based on valuable customer insights.

What this project contain ?

1. Custom user authentication system

Although django provide built in user authentication system. But in this project i have manupulated that user model into custom user model for this project. I added custom fields and methods in user model. for example date of birth, gender, etc.

User can register, login and see profile details. Profile details can be updated. E-mail verification is also there in my project through otp.

2. Store (Restaurant, Dishes and Reviews)

Store is the section of my website. Any user can visit to store section and watch restaurant, dishes (with price and details) and reviews.

3. Review Token

User can give review to any restaurant and their dishes. Restaurant owner provide a token (through E-mail) for review their dishes and restaurant. After that user can give review through pending review token section of my website.

4. Restaurant and Dishes

Those users who are owner of any restaurant can add, edit and delete dishes to that restaurant. If restaurant owner have multiple branches in different cities. He can handle with only one account for each city. They can also add and delete multiple dishes to multiple restaurant. With not only that owner can use token based review system to take feedback and rating from customers.

They can mention that either shop is open or close with reason provided.

Summary

This is the simple django framework based CRUD project. This enhances my learning towards software development. I learnt functionalities of different apps work together, the logic building and the way software developer works.

I have used tailwind and flowbite for writing frontend part of this project. Along with that i used vanilla javascript for writing frontend logics. XHtttp request is used for fetching data from backend through javascript

For Database, I love sqlite3. But wait, it is lightweight database with least functionalities written in 'C' language. So i used mysql for storing data. For development purpose, offcourse you can use sqlite3 database. But preference is yours to use. Django supports SQL database (mysql, postgresql, microsoft server) and avoid to use NON-SQL database.

This project is an open source project. You can check project github repository for more details.
